==

Connecting GitHub to Visual Studio Code

  • Open Visual Studio Code Terminal and check if git is installed: git --version
  • Configure git
    • git config --global user.name "Your Name"
    • git config --global user.email "your-email@example.com"
  • Generate an SSH key (Optional for Authentication) - recommended for secure authentication
    • Generate SSH Key: ssh-keygen -t rsa -b 4096 -C "your-email@example.com". Press Enter to save in default location and set a passphrase or leave it blank.
      • Copy the SSH key to Git Hub
        • Run cat ~/.ssh/id_rsa.pub
        • Copy the displayed key and go to GitHub → Settings → SSH and GPG keys → New SSH key. Paste the copied key and save it.
  • Open VS Code Click on Source Control (Cmd+Shift+G on mac) and sign in with Github.

Create a New Repository and Push to GitHub

  • Use terminal and navigate tot he folder of the project
  • Initialise Git: git init
  • Add files to Git: git add . (this may take a while)
  • Commit the files: git commit -m "First commit"
  • Connect the local repo to GitHub: git remote add origin https://github.com/your-username/your-repo.git
  • Push the code: git push -u origin main
  • If using SSH, replace the https:// URL with git@github.com:your-username/your-repo.git.

Clone an Existing GitHub Repository

  • Open the Comman Pallete (Cmd + Shift + P on mac)
  • Type and select Git:Clone
  • Paste the repository URL from GitHub
  • Choose a local folder to save the repository.
  • Open the cloned folder in VS Code.

Pull and Push Changes

  • Pull the latest changes from GitHub: git pull origin main
  • Push the new changes:
    • git add .
    • git commit -m "Updated changes"
    • git push origin main

Merge Conflicts

  • New computer must pull the latest changes from Github: git pull origin main
  • Resolve merge conflicts (if any)
    • If there are no conflicts, Git will automatically merge the changes
    • If conflict exists, Git will mark conflicting files and you'll need to manually resolve them in VS Code
    • Open the conflicting files in VS Code and choose which changes to keep.
    • After resolve the conflict, add and commit
      • git add .
      • git commit -m "Resolved merge conflicts"
    • Push the updated code to GitHub
      • git push origin main
  • Rebase is another method
    • Fetch the latest changes without merging: git fetch origin
    • Rebase your local changes on top of the latest remote changes: git rebase origin/main. If there are conflicts, Git will stop and ask you to resolve them.
    • After resolving, continue rebase with: git rebase --continue
    • Push the rebased changes: git push origin main
